## Why

当前导出流程在服务端使用固定的默认样式配置与固定的缩放级别范围，无法直接复用 Web 端样式编辑器导出的样式表，也无法通过简单参数控制导出的缩放级别区间。实际业务中常见需求包括：在 Web 端调好样式后“一键批量导出”对应的 XYZ 影像瓦片，以及按不同场景导出不同级别范围（如仅导出 7–14 级以控制磁盘占用与导出耗时）。因此，需要一次性打通“Web 样式 → 服务端导出”的链路，并提供可配置的缩放级别范围。

## What Changes

- 接口层支持传入 Web 端导出的样式配置（JSON），服务端直接使用该样式渲染导出。
- 约定并实现默认导出缩放级别为 0–17 级，满足完整底图级别覆盖的常见需求。
- 在导出参数中增加可配置的 `minZoom` / `maxZoom` 字段，允许调用方收窄导出级别范围（如 7–14），以平衡渲染时间与存储成本。
- 提供面向开发者的命令行脚本/调用示例，简化“传入样式 + 配置级别范围 + 启动导出”的操作流程。
- 为后续将该导出能力集成进 Web 端留出接口与参数设计空间（本次仅完成服务端与脚本形态，不做 UI 集成）。

## Capabilities

### New Capabilities
- `style-driven-imagery-export`: 允许通过 Web 端导出的样式表驱动服务端批量渲染 XYZ 影像瓦片，并支持可配置的缩放级别范围（包含默认 0–17 级与任意子区间）。

### Modified Capabilities
- `vector-tiles-to-cesium-imagery-export`: 在既有矢量瓦片转 Cesium 影像导出的能力上，扩展输入样式来源（支持外部样式 JSON）与缩放级别范围配置，但不改变已有导出任务创建/查询 API 的基本契约。

## Impact

- 影响 `VectorTileToCesiumImageryLayerTiles` 子项目下的导出服务实现，包括导出参数结构、样式加载逻辑与任务入参校验。
- 影响服务端示例脚本（如 `service/scripts/export-demo.sh`），需要更新为通过参数或输入样式文件驱动导出，并演示缩放级别范围配置与默认 0–17 级导出用法。
- 影响项目文档（`VectorTileToCesiumImageryLayerTiles/README.md` 及相关说明），需要补充“如何从 Web 导出样式并传给导出服务”“如何配置缩放级别范围”和“推荐的命令行调用方式”等内容。
- 为后续 Web 端集成留出接口与参数规范基础，但本次不改动 Web 前端 UI，实现仍集中在服务端与开发者指令层面。
