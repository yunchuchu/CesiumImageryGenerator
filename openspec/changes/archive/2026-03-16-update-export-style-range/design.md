## Context

当前 `VectorTileToCesiumImageryLayerTiles` 的导出服务通过固定的默认样式配置与固定的缩放级别范围触发批量渲染，示例脚本 `export-demo.sh` 直接读取共享目录下的 `default-style-config.json`，并在脚本内写死 `minZoom` / `maxZoom` 与导出范围 `bounds`。与此同时，Web 端已经具备样式编辑与 JSON 导出能力，但导出的样式暂未串联到服务端导出流程中，调用方无法方便地将 Web 保存的样式用于批量生成 XYZ 影像瓦片，也无法通过简单参数切换导出的缩放级别区间。

## Goals / Non-Goals

**Goals:**

- 允许导出服务接受来自 Web 端导出的样式 JSON（或样式文件路径），服务端直接使用该样式进行渲染。
- 约定并实现“默认导出 0–17 级”的行为，同时支持通过参数配置任意子区间（如 7–14）。
- 在不破坏现有 API 路径的前提下，明确导出请求体结构（包含样式与缩放级别范围），并提供一套易用的命令行脚本用法供开发者复用。
- 为未来将导出能力集成回 Web 前端预留稳定的参数契约。

**Non-Goals:**

- 不在本次改动中实现 Web 前端 UI 内的一键导出按钮或任务管理界面。
- 不改造底层矢量瓦片服务或渲染引擎，仅在现有导出服务能力之上扩展输入样式与缩放级别配置。
- 不引入新的队列系统或复杂的任务调度，仅保持当前最小实现。

## Decisions

- **请求体结构保持 `style` + `export` 两层**：沿用当前导出 API 中的 `style`（样式配置）与 `export`（导出参数）结构，在 `style` 中直接传入 Web 端导出的样式 JSON，在 `export` 中增加/规范 `minZoom` / `maxZoom` 字段含义，避免新增多余的嵌套层级。
- **默认缩放级别为 0–17**：在服务端实现中，当调用方未显式提供 `minZoom` / `maxZoom` 时，统一回退到 `0` 与 `17`，而非通过脚本写死，以保证 API 语义清晰且脚本示例仅作为便利入口。
- **导出脚本参数化**：将现有 `export-demo.sh` 从“完全写死配置的示例脚本”调整为“可接收样式文件路径与缩放级别区间的指令封装”，例如：`export-demo.sh path/to/style.json 7 14`，并在无参数时回退到“默认样式 + 0–17 级”。
- **保持 API 向后兼容**：`POST /api/exports` 的路径与基础行为不变，旧有调用者若仍按原请求体结构传参，只要满足最小必需字段，服务应继续工作，仅在缺省时按新的默认缩放级别策略处理。

## Risks / Trade-offs

- **样式兼容性风险**：Web 端导出的样式 JSON 可能与服务端期望的样式结构略有差异（字段命名或默认值），需要在样式加载/校验阶段增加合理的容错与错误提示，否则可能出现“本地预览正常、服务端导出失败”的体验问题。
- **磁盘与时间成本风险**：默认导出 0–17 级在某些区域可能产生较大的瓦片数量与存储占用，需要在文档与脚本中明确提醒使用者谨慎选择导出范围，并推荐根据实际业务设置合适的 `minZoom` / `maxZoom`。
- **接口语义复杂度增加**：在保持向后兼容的前提下扩展字段，可能带来一定的参数组合复杂度，需要通过清晰的命名与文档示例降低误用概率。
